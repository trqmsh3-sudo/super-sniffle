# 🚀 Complete Work Plan - TruthfulProducts

## 📋 Overview

A US-focused product research platform with two core tools:
1. **Product Intelligence** - Deep research on specific products
2. **Product Finder** - Find perfect products based on needs

---

## 🏗️ Technical Architecture

```
Frontend (React + Tailwind)
    ↓
Backend API (Node.js/Express)
    ↓
┌─────────────┬──────────────┬─────────────┐
│ Web Scraper │  AI Engine   │   Cache     │
│  Module     │  (Llama/API) │  (Redis)    │
└─────────────┴──────────────┴─────────────┘
    ↓           ↓              ↓
Data Sources: Amazon, Reddit, YouTube, Walmart, Best Buy
```

## 📊 Database Schema

### **PostgreSQL Tables:**

```sql
-- products table (cache product data)
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    product_name VARCHAR(500) NOT NULL,
    normalized_name VARCHAR(500) NOT NULL,
    category VARCHAR(100),
    data JSONB NOT NULL,
    last_scraped TIMESTAMP DEFAULT NOW(),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- search_history table
CREATE TABLE search_history (
    id SERIAL PRIMARY KEY,
    query VARCHAR(500) NOT NULL,
    product_id INTEGER REFERENCES products(id),
    user_ip VARCHAR(45),
    created_at TIMESTAMP DEFAULT NOW()
);

-- cached_results table
CREATE TABLE cached_results (
    id SERIAL PRIMARY KEY,
    product_id INTEGER REFERENCES products(id),
    result_data JSONB NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);

-- user_preferences table (for Product Finder)
CREATE TABLE user_preferences (
    id SERIAL PRIMARY KEY,
    session_id VARCHAR(255),
    category VARCHAR(100),
    budget_min INTEGER,
    budget_max INTEGER,
    priorities JSONB,
    requirements JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);

-- recommendations table
CREATE TABLE recommendations (
    id SERIAL PRIMARY KEY,
    session_id VARCHAR(255),
    product_ids INTEGER[],
    match_scores FLOAT[],
    created_at TIMESTAMP DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_products_normalized ON products(normalized_name);
CREATE INDEX idx_search_history_query ON search_history(query);
CREATE INDEX idx_cached_expires ON cached_results(expires_at);
```

### **Redis Cache Structure:**

```
# Product data cache (6 hours)
product:intel:{product_name} -> JSON

# Search suggestions cache (24 hours)
suggestions:{partial_query} -> Array

# Rate limiting
ratelimit:{ip}:{endpoint} -> Counter

# Active scraping jobs
scraping:job:{product_name} -> Status
```

## 🎯 API Endpoints

### **Product Intelligence:**

```
POST   /api/v1/product-intel/search
       Body: { "query": "Weber Spirit II E-310" }
       Returns: Full product analysis

GET    /api/v1/product-intel/suggestions
       Query: ?q=web
       Returns: Autocomplete suggestions

GET    /api/v1/product-intel/:productId
       Returns: Cached product data
```

### **Product Finder:**

```
POST   /api/v1/product-finder/start
       Body: { "category": "grills" }
       Returns: Session ID & first question

POST   /api/v1/product-finder/answer
       Body: { 
         "sessionId": "xxx",
         "question": "budget",
         "answer": { "min": 300, "max": 600 }
       }
       Returns: Next question or recommendations

GET    /api/v1/product-finder/recommendations/:sessionId
       Returns: Top 3-5 product matches

POST   /api/v1/product-finder/compare
       Body: { "productIds": [1, 2, 3] }
       Returns: Side-by-side comparison
```

### **Utility:**

```
GET    /api/v1/health
       Returns: Service status

GET    /api/v1/categories
       Returns: Available product categories

GET    /api/v1/stats
       Returns: Platform statistics
```

## 🔄 Data Flow

### **Product Intelligence Flow:**

```
1. User enters: "Weber Spirit II E-310"
   ↓
2. Frontend → POST /api/v1/product-intel/search
   ↓
3. Backend checks Redis cache
   ├─ Cache HIT → Return immediately
   └─ Cache MISS → Continue
   ↓
4. Fuzzy search correction
   "webber sprit" → "Weber Spirit II E-310"
   ↓
5. Parallel scraping (Promise.all):
   ├─ Amazon API/Scraper
   ├─ Reddit API
   ├─ YouTube API
   ├─ Walmart API
   ├─ Best Buy API
   └─ Home Depot Scraper
   ↓
6. Aggregate all data
   ↓
7. Send to AI Engine:
   {
     reviews: [...],
     prices: [...],
     sources: [...]
   }
   ↓
8. AI analyzes and generates:
   - Overall score
   - Pros/cons
   - Summary
   - Price analysis
   - Recommendation
   ↓
9. Save to PostgreSQL & Redis
   ↓
10. Return to frontend
   ↓
11. Display beautiful results!
```

### **Product Finder Flow:**

```
1. User clicks "Find My Perfect Product"
   ↓
2. Frontend → POST /api/v1/product-finder/start
   Body: { category: "grills" }
   ↓
3. Backend creates session, returns first question
   ↓
4. User answers questions (budget, priorities, etc.)
   ↓
5. Each answer: POST /api/v1/product-finder/answer
   ↓
6. After all questions:
   Backend searches database for matching products
   ↓
7. AI ranks products by fit score
   ↓
8. Return top 3-5 recommendations
   ↓
9. Display with "Why it matches" explanations
```

## ⚡ Performance Optimizations

### **Caching Strategy:**
```javascript
// Three-tier caching:

// 1. Memory cache (fastest, 5 min)
const memCache = new Map();

// 2. Redis cache (fast, 6 hours)
await redis.setex(key, 21600, data);

// 3. PostgreSQL (permanent)
await Product.create({ ... });
```

### **Parallel Scraping:**
```javascript
// Scrape all sources simultaneously
const [amazon, reddit, youtube, walmart] = await Promise.allSettled([
  amazonScraper(product),
  redditScraper(product),
  youtubeScraper(product),
  walmartScraper(product)
]);

// Continue even if one fails
const validData = results
  .filter(r => r.status === 'fulfilled')
  .map(r => r.value);
```

### **Request Batching:**
```javascript
// Batch multiple AI requests
const queue = [];
setInterval(async () => {
  if (queue.length > 0) {
    const batch = queue.splice(0, 10);
    await processAIBatch(batch);
  }
}, 1000);
```
